## Summary

Add a −6 dB/oct high-pass filter to each voice, placed between the source mix
and the ladder VCF (JP-8 topology: Source Mixer → HPF → VCF → VCA). VXN1
currently has no high-pass at all; this is the cheapest way to thin body and
shape acoustic-style patches.

## Acceptance criteria

- [x] New `crates/vxn-dsp/src/hpf.rs`: a one-pole high-pass kernel and a
      16-voice structure-of-arrays `PolyHpf` mirroring `PolyLadder`'s shape
      (`set_cutoff(v, …)` / `process(&in, &mut out)`), re-exported from the
      crate root.
- [x] `PolyHpf` integrated in `vxn-engine` `VoiceBank::render_block`: filter the
      `mix` lane through the HPF before it enters `ladder.process`. Runs at the
      oversampled rate alongside the ladder.
- [x] New param `HpfCutoff` (Hz, log, ~20–18000, default 20 ≈ "off"/fully open
      at the low end), **appended at the end of the `ParamId` table** to keep
      existing CLAP ids stable; add `hpf_cutoff` to `BlockCtx` and populate it
      in `build_ctx`.
- [x] `PolyHpf` state resets in `VoiceBank::reset_all` / `set_sample_rate`.
- [x] Tests: HPF attenuates DC and low frequencies and passes highs ~unchanged;
      at the default low cutoff the signal is essentially untouched; the
      existing param-table tests (`table_len_matches_count`,
      `ids_match_table_indices`, `defaults_in_range`) still pass.

## Notes

- Per-voice, not a global post-mix stage — the ladder is already per-voice
  (`PolyLadder`), so the HPF belongs in the same SoA loop.
- One pole is enough to match the JP-8's −6 dB/oct; no oversampling concerns
  beyond running at the same rate as the ladder.
- Appending the param at the end avoids disturbing `MATRIX_BASE` and the
  source-major/dest-minor matrix layout (the matrix params sit mid-enum).
- Validation: `cargo test -p vxn-dsp -p vxn-engine`.
