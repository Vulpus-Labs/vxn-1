## Summary

Final stage of the kernel: master tune + master volume + the assembled
CLAP-facing parameter table that every prior ticket has been writing into.
This is the surface the CLAP shell (later epic) and the production UI bind
against.

No FX is added here ‚Äî delay (0010) and reverb (0011) sit between the voice
sum and master out.

## Acceptance criteria

- [ ] `master_tune` adds cents to every voice's pitch (applied at the voice
      level, summed with bend + glide).
- [ ] `master_volume` is the final output gain. Range ‚àí60..+6 dB, exponential
      taper.
- [ ] Output limiter explicitly NOT implemented (per ADR ¬ß10 review and the
      Master panel decision).
- [ ] `vxn2-engine::params::ParamTable` enumerates every CLAP-automatable
      parameter from `PARAMETERS.md`:
      - 126 per-op (6 √ó 21)
      - 1 algo
      - 4 LFO1
      - 5 LFO2
      - 9 Pitch EG
      - 5 Mod Env
      - 6 Voice + assignment
      - 5 Stacking
      - 6 Delay
      - 5 Reverb
      - 2 Master
      = 174 total automatable
- [ ] Each `ParamDescriptor` carries: id (stable string, kebab-case),
      display name, plain range, normalised range (always [0, 1]), default,
      unit string, plain‚Üînormalised converters.
- [ ] Parameter writes thread-safely apply to the engine state (use the
      atomic / SPSC pattern VXN1 ships in `vxn-engine::SharedParams`; copy
      shape, not code).
- [ ] Mod matrix slots are NOT in `ParamTable` ‚Äî they're patch state,
      serialised separately. The matrix table is mutable from patch loads
      and the matrix overlay UI but not from CLAP automation in v1.
- [ ] Integration test: a stub host (in-process) sends note-on, sets each
      parameter through its full range, renders 1 second of audio per
      setting, asserts no NaN / Inf / silence-where-not-expected.
- [ ] Bench: `master_chain_full` (16 voices √ó density 4 √ó full FX chain).

## Notes

Stable parameter IDs are no longer load-bearing for VXN1's preset format
(per memory: `vxn1-id-stability-dropped`) ‚Äî VXN2 inherits this. Use kebab-
case string IDs (`op1-ratio`, `lfo2-delay`) but treat them as freely
renameable; preset format is name-keyed.

The 174-param table is large enough that you don't want to manually
enumerate it in a single match arm. Use a const slice of `ParamDescriptor`s
generated by a macro (or by a build script ‚Äî `vxn-engine` does the
const-slice route, copy that pattern).

Saturation: even without a limiter, output should not clip when individual
voices are reasonable but their sum exceeds 1.0. The master volume default
of ‚àí6 dB gives 6 dB of headroom for typical patches; patches that exceed
this are the user's responsibility (their DAW limits it).

Bench numbers from this ticket establish the kernel's headline CPU budget.
Document in the README once this lands.
