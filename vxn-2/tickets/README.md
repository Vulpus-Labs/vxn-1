# vxn-2 tickets

`open/` and `closed/` hold one markdown file per ticket, `NNNN-slug.md`.

**Numbering convention:** vxn-1 and vxn-2 keep *separate* ticket counters, both
starting at `0001`, so the numbers overlap historically (both projects have
0055–0060, etc.). A bare ticket number is always relative to its project — a
vxn-2 ticket number refers to `vxn-2/tickets/`, never `vxn-1/tickets/`. When
referencing across projects, qualify it (e.g. "vxn-1 0058"). New tickets take
the next free number in *this* project's sequence regardless of vxn-1.
